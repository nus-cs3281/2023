Vue LifeCycle Management

Description of the tool: Vue component life cycle hook

Aspect: Methods that can be used to create hook between Vue component and the template, enabling the pug file template to load information before rendering, and the Vue component to access information in the pug file template before and after its initial rendering.

created is a useful method for loading the data and retrieving the window hashes that are needed for the vue component after the component is created.

beforeMount is often used to access and modify the information of the component before the rendering.

mounted is called after the rendering and it is also used to access or modify the information of the component.

It is important to distinguish between create and beforeMount and mounted.

Related article:

https://medium.com/@akgarg007/vuejs-created-vs-mounted-life-cycle-hooks-74c522b9ceee

Documentation:

https://vuejs.org/v2/guide/instance.html#Instance-Lifecycle-Hooks

Tool/Technology 8

Git command for Merge commit statistics computation

Description of the tool: The git command to compute the statistics for a merge commit

Aspect: In each commit, what is stored is the snapshot of documents instead of differences from the previous commit. This poses challenges when computing statistics for conflict resolution of a merge commit. The command git log --cc --numstat <MERGE_COMMIT_HASH> will help to calculate the number of additions and deletions of a merge commit with respect to its parents. Note that this command only calculates the numbers for the files containing changes from both of the parent commits. Additionally, although not mentioned in documentation, the second related post suggests that the numbers are for the differences between the merge commit and its second parent, not the first parent. The first parent commit usually comes from the branch accepting the changes to be merged. Moreover, the arguments supplied in this command is important. Any diff related arguments such as --stat and --numstat in git log command will be ignored without an additional argument such as -c, --cc, and -m that indicates how to handle differences with multiple parent commits appropriately.

Related post:

https://stackoverflow.com/questions/57333971/git-log-numstat-has-weird-data

https://stackoverflow.com/questions/37036277/git-show-vs-git-log-numstat-output-difference

Documentation:

https://git-scm.com/docs/git-log

https://git-scm.com/docs/git-diff-tree
